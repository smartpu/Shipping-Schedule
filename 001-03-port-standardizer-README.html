工具简介

港口标准化工具用于将 001-02 解析生成的 CSV 文件中的港口名称标准化为统一格式 [英文名|代码|区域]。工具基于港口列表进行智能匹配，支持多种港口名称格式的识别和转换，并生成详细的转换报告，便于改进代码或手动处理未匹配的港口。

使用步骤

  1. 准备文件：确保已使用 001-02 工具解析生成 CSV 文件
  2. 载入文件：
    • 在本地文件系统环境下，可先选择港口列表文件（可选，推荐以获得完整映射）
    • 点击"载入 CSV 文件"按钮，选择要处理的 CSV 文件
  3. 查看报告：处理完成后，查看转换报告了解匹配情况
  4. 下载结果：点击"下载标准化后的 CSV"按钮保存处理结果

核心功能

  • ✅ 智能匹配：支持多种港口名称格式的识别和匹配
    • 精确匹配：直接匹配港口名称（支持大小写不敏感）
    • 去除方向词：自动去除"东西南北"等方向词后匹配
    • 括号提取：从 QINGDAO(青岛) 格式中提取英文或中文部分匹配
    • 逗号分隔：从 LOS ANGELES,CA 格式中提取逗号前的部分匹配
    • 模糊匹配：使用包含关系进行模糊匹配
  • ✅ 转换报告：生成详细的转换报告，包括：
    • 总体统计：显示处理的总行数
    • 起运港匹配情况：精确匹配、模糊匹配、未匹配的数量和列表
    • 目的港匹配情况：精确匹配、模糊匹配、未匹配的数量和列表
  • ✅ 批量处理：一次性处理整个 CSV 文件的所有行
  • ✅ Excel兼容：生成的 CSV 文件使用 UTF-8 BOM 编码，Excel 可直接打开
  • ✅ 本地支持：在本地文件系统环境下，支持手动选择港口列表文件

匹配规则

工具会按照以下优先级进行港口匹配：

  1. 精确匹配：直接匹配港口名称（支持大小写不敏感）
  2. 去除方向词：去除"东西南北"等方向词后匹配
  3. 括号提取：从 QINGDAO(青岛) 格式中提取英文或中文部分匹配
  4. 逗号分隔：从 LOS ANGELES,CA 格式中提取逗号前的部分匹配
  5. 模糊匹配：使用包含关系进行模糊匹配

转换报告说明

转换报告包含以下信息：

  • 总体统计：显示处理的总行数
  • 起运港匹配情况：
    • 精确匹配：数量（绿色显示）
    • 模糊匹配：数量（黄色显示）
    • 未匹配：数量（红色显示）和未匹配的港口列表
  • 目的港匹配情况：
    • 精确匹配：数量（绿色显示）
    • 模糊匹配：数量（黄色显示）
    • 未匹配：数量（红色显示）和未匹配的港口列表

未匹配处理

如果出现未匹配的港口，可以：

  • 检查港口列表文件，确认港口是否在列表中
  • 手动编辑 CSV 文件，将未匹配的港口替换为标准化格式 [英文名|代码|区域]
  • 联系开发人员，将新的港口添加到港口列表中

注意事项

  • CSV 文件必须包含"启运港"和"目的港"列
  • 处理大文件时可能需要一些时间，请耐心等待
  • 下载的 CSV 文件使用 UTF-8 BOM 编码，确保 Excel 正确显示中文
  • 在本地文件系统环境下，建议手动选择港口列表文件以获得完整的映射关系
  • 如果未手动选择港口列表文件，工具会尝试使用已加载的数据，但可能不完整

常见问题

Q1: 为什么有些港口未匹配？

A：可能的原因：

  • 港口名称不在港口列表中
  • 港口名称格式与列表中的格式差异较大
  • 港口名称包含特殊字符或格式

可以查看转换报告中的未匹配列表，手动处理或联系开发人员添加新的港口。

Q2: 本地环境下如何获得完整的映射？

A：在本地文件系统环境下，工具会显示一个可选的港口列表文件上传区域。建议手动选择港口列表文件，这样可以获得完整的映射关系，提高匹配成功率。

Q3: CSV 文件在 Excel 中打开中文乱码

A：工具生成的 CSV 文件使用 UTF-8 带 BOM 编码，应该可以在 Excel 中正常打开。如果仍然出现乱码，请尝试：

  • 使用 Excel 的"数据" → "从文本/CSV"功能导入
  • 在导入时选择 UTF-8 编码
  • 或者使用其他支持 UTF-8 的软件（如 WPS、LibreOffice）打开

Q4: 处理速度很慢

A：处理速度取决于文件大小和港口数量。如果文件很大，请耐心等待。建议使用 Chrome 或 Edge 浏览器以获得最佳性能。
