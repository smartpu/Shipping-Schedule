Market Analysis · 市场分析 使用说明

工具简介

Market Analysis · 市场分析 是一款面向港口维度的航线运力与 AI 分析工具。通过一份包含 schedule 工作表的 Excel、可选的市场周报 PDF 以及 DeepSeek / KIMI / 通义千问 API Key，即可完成港口运力洞察、供需趋势可视化与 AI 决策建议输出。工具聚焦以下两大模块：

  1. Module 01 · 航线运力分析 (SAILING SCHEDULE PULSE)
  2. Module 02 · AI 趋势分析 (AI TREND ANALYSIS)

快速开始

前置准备

  1. 载入 Excel 数据源：文件需包含 schedule 工作表，并具备港口、航线、周别、运力、船名航次等字段。
  2. 载入市场周报：支持一次选择多份 PDF（如 Alphaliner Newsletter、Linerlytica Weekly 等），便于 AI 参考最新情报。
  3. 准备 AI 密钥：工具已集成临时 API Key（服务器配置），可直接使用。使用临时 API Key 时会弹出提示，强烈建议自行申请 API Key 自用（临时 API Key 随时可能过期），用户输入的 API Key 优先级高于配置文件中的临时 API Key。调用费用及合规责任由使用方承担。详细说明见下方"Module 02 · AI 趋势分析"部分。

基本流程

  1. 打开 001-04-market-analysis.html
  2. 顶部载入区：
    • 点击"选择Excel文件"，载入包含 schedule 工作表的 Excel
    • 点击"载入市场周报 (PDF)"（可选，可多选）
  3. 当 Excel 解析成功后，港口筛选、表格、图表和 AI 模块自动显示
  4. 根据需求完成筛选、AI 分析，必要时点击"导出整页 PDF"

Module 01 · 航线运力分析

  1. 使用港口多选筛选与航线搜索锁定目标市场。
  2. "全部 / 运力 / 派船"视图用于聚焦不同指标；表格支持即输即筛。
  3. 选择目的港后自动绘制堆积柱（运力）+ 折线（派船）趋势，便于观察当周+未来四周变化。

Module 02 · AI 趋势分析

  1. 填写"其他影响因素"表格、刷新燃油/指数数据，作为 AI 提示词的优先依据。
  2. 分别配置 DeepSeek / KIMI / 通义千问 API 后，可对比三种模型的洞察结果，支持导出 PDF 汇报。
  3. 模型选择建议：
    • DeepSeek：推荐 deepseek-chat（基础）或 deepseek-v3（高性能）
      申请 API Key：https://platform.deepseek.com
    • KIMI：推荐 moonshot-v1-32k（平衡），moonshot-v1-k2（K2新模型，需确认API可用）
      申请 API Key：https://platform.moonshot.cn/console/api-keys
    • 通义千问：推荐 qwen-max（最佳性能），qwen-turbo（经济型）
      申请 API Key：https://dashscope.console.aliyun.com/apiKey
  4. API Key 使用说明：
    • 工具已集成临时 API Key（服务器配置），可直接使用，无需手动输入
    • 使用临时 API Key 时会弹出提示：系统会提示您正在使用临时 API Key，并强烈建议自行申请自己的 API Key
    • 强烈建议自行申请 API Key：临时 API Key 随时可能过期无法使用，且可能存在配额限制。自行申请可获得更好的使用体验和更高的调用额度
    • 如果用户自行填写 API Key，用户输入的 API Key 优先级高于配置文件中的临时 API Key
    • 配置会自动保存到浏览器本地存储，下次打开会自动加载
    • AI 输出长度：已设置为 8000 tokens，可支持更长的分析输出
  5. 收费说明：各模型按调用次数/Token 计费，具体价格请参考各平台官网。建议先使用经济型模型测试，再根据需求选择高性能模型。

核心功能模块

Module 01 · 航线运力分析 (SAILING SCHEDULE PULSE)

定位：港口多选筛选 + 航线搜索 + 五周运力/派船趋势

主要功能

  • 港口多选筛选：仅需勾选一个或多个港口即可联动表格与趋势图
    • 智能合并：多选港口时，系统会自动按航线分组，同一船名航次在不同港口出现时会合并显示，避免重复统计
    • 例如：选择"洛杉矶"和"长滩"两个港口，如果同一航线（如 CIX2）在同一周有相同船名航次，会合并为一条记录
  • 航线搜索：在"共舱船公司"表头输入关键词即可实时过滤
  • 视图模式：全部 / 运力 / 派船 三种显示模式，快速聚焦指标
  • 周别统计：自动提取"当周 + 未来四周"（共5周），表头显示 YYYY/WW + 日期范围
    • 数据抓取原则为开船前，因此不显示上周数据
  • 趋势图：堆积柱（运力） + 折线（派船），带双 Y 轴便于对比
  • 明细提示：悬停表格单元格查看船名航次（K列）、运力TEU（L列）、开航时间（M列）
  • 汇总行：自动累计所选港口下的每周运力与派船数，便于总结

Module 02 · AI 趋势分析 (AI TREND ANALYSIS)

定位：DeepSeek × KIMI × 通义千问 三模型交叉验证，输出涨/维/降建议和九大项分析

主要能力

  • 其他影响因素输入：收货情况、码头情况、额外运力、市场运费、其他事件
  • 外部数据抓取：
    • Ship & Bunker：VLSFO / MGO / IFO380（含 WoW）
    • CCFI：综合 + 12 条航线指数（含 WoW）
    • WCI：全球综合 + 8 条航线现货价（含 WoW）
    • FBX：FBX 综合 + 13 条航线指数（含 WoW）
  • AI 分析输出：本周结论、需求/供给/突发/内部策略/指数解读、风险提示、下周展望等九大板块
  • 三模型配置：
    • DeepSeek：deepseek-chat（默认）及 deepseek-v3、deepseek-reasoner、deepseek-r1、deepseek-coder 等模型
    • KIMI (Moonshot)：moonshot-v1-32k（默认）及 moonshot-v1-8k、moonshot-v1-128k、moonshot-v1-k2 等模型
    • 通义千问 (Qwen)：qwen-max（默认，推荐）及 qwen-turbo、qwen-plus、qwen-max-longcontext 等模型
    • API Key、URL、模型选择都会保存在浏览器 localStorage

触发条件

  1. 至少选择一个港口并加载出运力数据
  2. 已配置有效 API Key
  3. 点击"使用 DeepSeek 分析"、"使用 KIMI 分析"或"使用通义千问分析"

数据格式说明

Excel（schedule 工作表）

Excel 列 内容 列定位 说明
港口目的港名称B 列纯文本，如 BALBOA (巴尔博亚)
航线ID航线标识C 列例如 ROUTE001
航线备注航线备注信息H 列纯文本，如 美西快线
共舱船公司航线/船公司名称G 列如 OCEAN ALLIANCE
船名/航次船名与航次K 列例如 MSC OSCAR/123W
运力 (TEU)集装箱位L 列数值，如 14000
船期 (开船日)开船日期M 列日期字符串
周别YYYYWW 或"2025年第47周"R 列当周+未来四周

⚠️ 列必须保持在指定列号（B/C/G/H/K/L/M/R），如需调整请同步修改代码映射。

周别规则

  • 支持格式：202547、2025年第47周、2025-47、2025/47
  • 周定义：周日 → 周六，例如 2025/11/16 ~ 2025/11/22 为 2025/47
  • 跨年周别合并规则：如果遇到跨年周别（如 2025/53 和 2026/01），且两个周别加起来正好是 7 天，AI 分析时会自动将这两个周别的数据合并计算为一周，避免误判为运力短缺。表格和图表中会分别显示这两个周别的数据，但在 AI 分析时会合并计算。

常见问题

Excel 加载失败？

  • 确认文件中存在 schedule 工作表
  • 列顺序是否保持 B/C/G/I/L/M/R
  • 浏览器 Console 是否有错误

港口筛选没显示？

  • Excel 解析失败或列名不匹配
  • 刷新页面重新载入

AI 分析失败？

  • 如果使用的是临时 API Key（服务器配置），可能已过期或配额不足，建议自行申请 API Key
  • 使用临时 API Key 时会弹出提示对话框，确认后即可继续使用
  • 如果已自行配置 API Key，请检查 API Key、URL 和模型 ID 是否正确
  • 确认网络可访问 DeepSeek / KIMI / 通义千问
  • 查看提示信息（如配额不足、接口错误）
  • 查看浏览器控制台是否有错误信息
