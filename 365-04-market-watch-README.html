Market Watch · 市场观察 使用说明

工具简介

Market Watch · 市场观察 是一款集成的海运市场分析工具，通过一份 Excel 文件即可联动运价趋势、航线供给与 AI 决策，轻松完成跨板块洞察。工具整合了运费走势可视化、航线运力分析和 AI 智能分析三大核心模块，为海运定价与航线决策提供数据支持。

快速开始

前置准备

  1. 载入 Excel 数据源：文件需同时包含 market / schedule 工作表，并具备区域、港口、航线、周别、运力、船名航次、20GP / 40GP 等字段。
  2. 载入市场周报：支持一次选择多份 PDF（如 Alphaliner Newsletter、Linerlytica Weekly 等），便于 AI 参考最新情报。
  3. AI 密钥：工具已集成临时 API Key（服务器配置），可直接使用。使用临时 API Key 时会弹出提示，强烈建议自行申请 API Key 自用（临时 API Key 随时可能过期），用户输入的 API Key 优先级高于配置文件中的临时 API Key。调用费用及合规责任由使用方承担。详细说明见下方"Module 03 · AI 趋势分析"部分。

基本使用流程

  1. 双击打开 365-04-market-watch.html
  2. 在顶部载入区域：
    • 点击"选择Excel文件"，选择包含 market 和 schedule 工作表的 Excel 文件
    • 点击"载入市场周报 (PDF)"，可一次选择多份 PDF 报告（可选）
  3. 等待数据加载完成后，三个模块的筛选控件将自动显示
  4. 根据需求使用各模块功能进行分析
  5. 如需导出报告，点击"导出整页 PDF"按钮

Module 01 · 运费走势图

  1. 多选区域与港口，系统会自动匹配 20GP / 40GP 报价并提示价格区间。
  2. 可设定开始/结束日期，查看特定周期的价格走势，并用提示文本快速复制摘要。

Module 02 · 航线运力分析

  1. 使用四级筛选（区域 → 子区域 → 国家/地区 → 港口）与航线搜索锁定目标市场。
  2. "全部 / 运力 / 派船"视图用于聚焦不同指标；表格支持即输即筛。
  3. 选择目的港后自动绘制堆积柱（运力）+ 折线（派船）趋势，便于观察当周+未来四周变化。

Module 03 · AI 趋势分析

  1. 填写"其他影响因素"表格、刷新燃油/指数数据，作为 AI 提示词的优先依据。
  2. 可对比三种模型的洞察结果，支持导出 PDF 汇报。工具已集成测试 API Key，可直接使用；如需自行配置，见下方"API Key 使用说明"。
  3. 模型选择建议：
    • DeepSeek：推荐 deepseek-chat（基础）或 deepseek-v3（高性能）
      申请 API Key：https://platform.deepseek.com
    • KIMI：推荐 moonshot-v1-32k（平衡），moonshot-v1-k2（K2新模型，需确认API可用）
      申请 API Key：https://platform.moonshot.cn/console/api-keys
    • 通义千问：推荐 qwen-max（最佳性能），qwen-turbo（经济型）
      申请 API Key：https://dashscope.console.aliyun.com/apiKey
  4. API Key 使用说明：
    • 工具已集成临时 API Key（服务器配置），可直接使用，无需手动输入
    • 使用临时 API Key 时会弹出提示：系统会提示您正在使用临时 API Key，并强烈建议自行申请自己的 API Key
    • 强烈建议自行申请 API Key：临时 API Key 随时可能过期无法使用，且可能存在配额限制。自行申请可获得更好的使用体验和更高的调用额度
    • 如果用户自行填写 API Key，用户输入的 API Key 优先级高于配置文件中的临时 API Key
    • 配置会自动保存到浏览器本地存储，下次打开会自动加载
    • AI 输出长度：已设置为 8000 tokens，可支持更长的分析输出
  5. 收费说明：各模型按调用次数/Token 计费，具体价格请参考各平台官网。建议先使用经济型模型测试，再根据需求选择高性能模型。

核心功能模块

Module 01 · 运费走势图 (RATE TRENDS CHART)

功能定位：多港口报价、区间提示与时段筛选，一屏掌握市场温度

主要功能

  • 多选筛选：
    • 支持多选区域和港口
    • 系统自动匹配 20GP / 40GP 报价
    • 自动显示每个选中港口的价格区间（格式：港口名：20GP 最小值 ~ 最大值，40GP 最小值 ~ 最大值）
  • 时间范围筛选：
    • 可设定开始时间和结束时间
    • 查看特定周期的价格走势
    • 时间控件与图表标题同行显示
  • 可视化展示：
    • 交互式折线图，清晰展示价格趋势
    • 同时显示 20GP（实线）和 40GP（虚线）价格走势
    • 支持多港口对比，不同港口使用不同颜色区分

Module 02 · 航线运力分析 (SAILING SCHEDULE PULSE)

功能定位：四级筛选联动运力、派船与趋势，精准锁定当周+未来四周供给

主要功能

  • 四级筛选体系：
    • 区域 → 子区域 → 国家/地区 → 港口
    • 支持多选，筛选条件逐级联动
    • 筛选后自动更新表格和图表
    • 智能合并：多选港口时，系统会自动按航线分组，同一船名航次在不同港口出现时会合并显示，避免重复统计
  • 航线搜索：在表头"共舱船公司"列输入关键词，支持中文输入，实时过滤
  • 三种视图模式：全部 / 运力 / 派船 三种显示模式，快速聚焦指标
  • 周别统计：固定显示当周+未来四周（共5周），表头显示 YYYY/WW + 日期范围
  • 趋势图表：选择区域后自动显示运力与派船趋势图，堆积柱形图展示各航线运力分布，折线图展示派船总数变化

Module 03 · AI 趋势分析 (AI TREND ANALYSIS)

功能定位：DeepSeek × KIMI × 通义千问 三模型交叉验证航线策略、供需与价格建议

主要功能

  • 三模型支持：
    • DeepSeek：支持 deepseek-chat、deepseek-v3、deepseek-reasoner、deepseek-r1、deepseek-coder 等模型
    • KIMI (Moonshot)：支持 moonshot-v1-8k、moonshot-v1-32k、moonshot-v1-128k、moonshot-v1-k2 等模型
    • 通义千问 (Qwen)：支持 qwen-turbo、qwen-plus、qwen-max、qwen-max-longcontext 等模型
    • 可切换对比不同模型的分析结论
    • 配置自动保存到浏览器本地存储
  • 其他影响因素输入：收货情况、码头情况、额外运力、市场运费、其他事件
  • 外部数据抓取：
    • Ship & Bunker：VLSFO / MGO / IFO380（含 WoW）
    • CCFI：综合 + 12 条航线指数（含 WoW）
    • WCI：全球综合 + 8 条航线现货价（含 WoW）
    • FBX：FBX 综合 + 13 条航线指数（含 WoW）

数据格式说明

Excel 文件结构

market 工作表

用于运费走势图模块，需包含以下列：

  • 日期列：日期格式（A列或包含"日期"、"date"、"时间"等关键词）
  • 目的港列：港口名称（D列或包含"港口"、"port"、"目的港"等关键词）
  • 区域列：区域分类（I列或包含"区域"、"region"、"地区"等关键词）
  • 20GP 列：20GP 价格（列名包含 "20GP"、"20gp"、"GP20"）
  • 40GP 列：40GP 价格（列名包含 "40GP"、"40gp"、"GP40"）

schedule 工作表

用于航线运力分析模块，需包含以下列：

  • 目的港列：港口名称（第2列或包含"港口"、"目的港"、"pod"等关键词）
  • 区域列：区域分类（包含"区域"、"大区"、"region"等关键词）
  • 子区域列：子区域分类（包含"子区域"、"次区域"、"sub"等关键词）
  • 国家/地区列：国家或地区名称（包含"国家地区"、"国家/地区"、"国家"、"country"等关键词）
  • 航线列：共舱船公司或航线名称（F列或包含"共舱"、"船公司"、"航线"、"route"等关键词）
  • 周别列：周别代码（N列或包含"周别"、"周次"、"周数"、"week"等关键词，格式如：202547 或 2025年第47周）
  • 运力列：运力（TEU）（O列或包含"运力"、"teu"、"舱位"等关键词）

周别规则

  • 支持格式：202547、2025年第47周、2025-47、2025/47
  • 周定义：周日 → 周六，例如 2025/11/16 ~ 2025/11/22 为 2025/47
  • 跨年周别合并规则：如果遇到跨年周别（如 2025/53 和 2026/01），且两个周别加起来正好是 7 天，AI 分析时会自动将这两个周别的数据合并计算为一周，避免误判为运力短缺。表格和图表中会分别显示这两个周别的数据，但在 AI 分析时会合并计算。

常见问题

Excel 文件加载失败怎么办？

请检查：

  • Excel 文件是否包含 market 和 schedule 两个工作表
  • 工作表名称是否正确（不区分大小写，但需包含关键词）
  • 数据列是否包含必要的关键词
  • 浏览器控制台是否有错误信息

AI 分析失败怎么办？

请检查：

  • 如果使用的是临时 API Key（服务器配置），可能已过期或配额不足，建议自行申请 API Key
  • 使用临时 API Key 时会弹出提示对话框，确认后即可继续使用
  • 如果已自行配置 API Key，请检查 API Key、URL 和模型 ID 是否正确
  • 网络连接是否正常，确保可以访问 AI API
  • API 配额是否充足
  • 是否选择了至少一个区域（第一级筛选）
  • 查看浏览器控制台是否有错误信息

PDF 导出失败怎么办？

请检查：

  • 网络连接是否正常（需要加载 html2canvas 和 jsPDF 库）
  • 浏览器是否支持相关功能
  • 页面内容是否过长（建议先收起使用说明）
